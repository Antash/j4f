Написать программу, которая реализует функцию поиска файлов на локальных дисках персонального компьютера.
 
Создаваемая программа должна удовлетворять следующим требования:
-Возможность указания параметров для поиска файлов (сделать разумный выбор по своему усмотрению)
-Возможность запуска нескольких асинхронных копий процесса поиска и отображения общего результата в одном общем окне.
-Возможность остановки процессов, просмотра их статуса, отображения количества найденных файлов (например).
-Возможность расширения параметров поиска с помощи подключаемых модулей (Plugins)
	-Реализовать модуль поиска по тегу внутри XML
	-Реализовать модуль поиска класса/интерфейса среди дотнетовских сборок.
-Устойчивость к ошибкам.
-Интерфейс – любой WPF, WinForms, Web
-Язык реализации – C#

--------------------
Замечания к условию:
--------------------

1. Фразу "Возможность запуска нескольких асинхронных копий процесса поиска"
Можно трактовать двояко: речь может идти как и о отдельных процессах,
так и о асинхронных потоках. Будем понимать под словом "процесс поиска" все же поток. 
Так как использование отдельных процессов для решения этой задачи избыточно
и не даст преимуществ над потоками, к тому же процессы чуть сложнее будет 
синхронизировать (придется использовать, например WCF named pipes),
а так же передавать результаты на хост будет накладно.

2. Будем для простоты считать, что .NET-сборки всегда будут храниться в файлах *.dll или *.exe, а xml-документы - в *.xml

3. Сравнение реализовано:
	-для xml-тега - проверкой на равенство;
	-для типов .net - регулярные выражения;
	-для файлов - работает стандартный searchPattern

4. Для предотвращения риска зацикливания алгоритма поиска на циклических ссылках 
в файловой системе было принято решение не использовать P-Invoke, а обойтись
менее эффективным, нo более компактным способом.
